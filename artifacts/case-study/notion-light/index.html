How I built a Claude Code agent that reads a Notion source entry and produces five platform-ready content drafts — with linked to-dos — in under 2 minutes.

Yingshi Liu May 2026 Claude Code + Python + Notion API Open source
The Problem

One insight. Hours of manual work.

I capture ideas constantly — in Notion, in articles I read, in podcasts I listen to. The bottleneck isn't having insights. It's turning them into publishable content across multiple channels without losing hours to reformatting, task creation, and context switching.

① Draft the angle for the audience
② Write the full post body
③ Reformat for each platform
④ Create and link publish tasks

30–60 min. Per idea. Per channel.

The System

A Claude Code agent as the creative engine.

Signal to Asset is a Claude Code agent that lives in a project directory. You paste a Notion URL into the chat. The agent fetches the entry, asks for your hook and timeline, then writes five content drafts and saves everything — assets and to-dos — to Notion via the API.

Three Python scripts handle all API calls. Claude handles all writing. There's a human approval gate before anything is saved, so you stay in control.

How It Works

Step by step, from URL to ready-to-review.

The agent runs three steps: fetch the source entry, draft all five assets, then save them with linked to-do tasks. Each step is a dedicated Python script that handles caching, property mapping, and Notion API calls.

Stack & Links

Built to be simple, not impressive.

The goal was to remove friction, not add infrastructure. No database, no scheduler, no deployment. Just Python scripts, the Notion API, and Claude Code as the runtime. Anyone can clone the repo, fill in a .env file, and run it.
